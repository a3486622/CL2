# 哄哄模拟器 - 产品规格说明书

## 1. 问题陈述

### 1.1 产品定位
情侣吵架后不知道怎么哄对方的练习工具，通过游戏化方式帮助用户学会哄人技巧。

### 1.2 目标用户
- 核心用户：觉得好玩、当社交货币玩的人群
- 核心需求：娱乐性 + 分享性
- 次要需求：在娱乐中潜移默化学会哄人

### 1.3 分享动机
用户玩完后想分享的场景：
1. 晒自己情商高/运气好，"10轮以内过关，我是不是很厉害？"
2. 晒自己翻车搞笑，故意选奇葩选项看AI怎么骂自己（自嘲式分享）
3. 晒聊天记录本身很有趣，像看段子一样

---

## 2. 方案描述

### 2.1 核心玩法
AI扮演正在生气的对象，用户通过选择题的方式回应，在10轮内把对方哄好。

### 2.2 游戏流程

```
1. 用户选择对方性别（女朋友/男朋友）
2. 用户从预设场景中选一个，进入游戏
3. AI生成对方说的第一句话 + 6个回复选项
4. 用户选一个选项
5. 根据选项类型显示反应等待动画（加分2秒/普通减分1秒/奇葩3秒）
6. AI根据选择动态生成下一轮对话 + 新的6个选项
7. 如此循环，直到游戏结束
8. 游戏结束：展示结果 + 复盘 + 成就 + 分享卡片
```

### 2.3 预设场景

| 场景 | 描述 |
|-----|------|
| 忘记纪念日 | 今天是你们在一起三周年，你完全忘了 |
| 深夜不回消息 | 你昨晚打游戏到凌晨三点，对方发了十几条消息你都没回 |
| 被发现和异性聊天 | 对方看到你和异性朋友的暧昧聊天记录 |
| 把对方的猫弄丢了 | 你帮对方照顾猫的时候，猫跑丢了 |
| 当众让对方没面子 | 你在朋友聚会上开了一个过分的玩笑 |

### 2.4 好感度系统

| 属性 | 值 |
|-----|---|
| 初始值 | 20分 |
| 满分 | 100分 |
| 最低值 | -50分 |
| 胜利条件 | 10轮内好感度 >= 80 |
| 失败条件 | 好感度降到 -50 以下，或10轮用完好感度仍 < 80 |

**好感度展示规则**：
- 不显示具体加减数字
- 只展示当前总分进度条 + 方向提示（+ 或 -）
- 方向提示有动画效果（加分绿色上升，减分红色下降）

### 2.5 选项设计

**每轮6个选项分布**：
- 2个加分选项（+5 到 +20）：真诚道歉/具体弥补方案/提起共同回忆等
- 4个减分选项（-5 到 -30）：
  - 1-2个普通减分：敷衍、转移话题、找借口
  - 2-3个奇葩搞笑选项：离谱到好笑

**奇葩选项红线（禁止出现）**：
- 外貌羞辱（"你生气的样子真丑"）
- 提及分手/离婚（"那就分手"）
- 提及对方家人（"都怪你妈"）

**选项交互**：
- 一次性展示全部6个选项
- 选项顺序随机打乱，不固定加分项位置
- 选项超过一行需换行显示

### 2.6 反应等待机制

用户选择后，根据选项类型显示等待动画：

| 选项类型 | 等待时长 | 动画效果 |
|---------|---------|---------|
| 加分选项 | 2秒 | 对方头像思考中 |
| 普通减分选项 | 1秒 | 对方头像轻微皱眉 |
| 奇葩选项 | 3秒 | 对方头像沉默/空气安静 |

### 2.7 回退机制

用户可以回到上一轮重新选择：
- 每次选择后自动创建存档点
- 提供"回退"按钮（限用一次？）
- 回退后重新生成6个不同的选项
- 本局已出现过的选项不允许重复出现

### 2.8 动态生成规则

**LLM生成要求**：
- 不预设题库，每一轮通过LLM动态生成
- 对方说的话必须和前面的对话连贯，模拟同一段连续对话
- 不能出现重复的题目和选项
- 已出现过的选项需记录，避免重复

**情绪变化规则**：

| 好感度区间 | 对方情绪状态 |
|-----------|-------------|
| -50 ~ 0 | 非常生气，冷暴力或激烈质问 |
| 0 ~ 30 | 还在生气，但愿意听你说 |
| 30 ~ 60 | 开始软化，嘴上生气但语气缓和 |
| 60 ~ 80 | 快被哄好了，可能撒娇或小声说"哼" |
| 80+ | 原谅了，但还要你保证不再犯 |

### 2.9 界面设计

**整体风格**：微信私聊界面

**布局要素**：
- 顶部：好感度进度条 + 当前轮次（第3轮/共10轮）
- 中间：对话区域
  - 对方的话在左侧，带头像
  - 用户选择的话在右侧，带头像
- 底部：6个选项按钮

**头像系统**：
- 对方头像：卡通形象，5种情绪状态（对应5个好感度区间）
- 用户头像：简单风格

**动画效果**：
- 好感度变化：方向提示（+ / -）绿色/红色上升
- 消息出现：打字机效果（流式输出）
- 成就解锁：徽章弹出动画

### 2.10 语音功能（TTS）

**声音选择**：
- 女朋友默认：温柔女声/霸道御姐/可爱软妹
- 男朋友默认：低沉男声/温柔男声
- 默认绑定性别，但允许用户切换

**语音规则**：
- 对方每一轮回复自动生成语音
- 页面上有播放按钮可手动播放
- 语音需体现情绪变化

### 2.11 游戏结束

**胜利（好感度 >= 80）**：
- 撒花动画
- 对方说一句甜蜜的话 + 语音播放
- 显示通关评价（好感度级别 + 轮数级别）
- 成就展示
- 复盘总结
- 分享卡片生成

**失败（好感度 < 80 或 <= -50）**：
- 心碎动画
- 对方说一句绝情的话 + 语音播放
- 显示失败原因
- 成就展示
- 复盘总结
- "再试一次？"按钮

### 2.12 成就系统

**成就解锁方式**：条件达成时随机抽取一个名字

**成就列表**：

| 触发条件 | 候选名字 |
|---------|---------|
| 零翻车（10轮全加分） | 嘴甜本甜、天花乱坠、社牛附体 |
| 搞笑担当（3次+奇葩选项） | 整活大师、小丑竟是我自己、欢乐喜剧人 |
| 中规中矩（5次+普通减分） | 普通市民、及格型选手、平庸之才 |
| 情商王者（5轮内+好感96+） | 人精本精、读心术士、人形灭火器 |
| 极限翻盘（0以下翻到通关） | 逆风翻盘、绝地求生、真香定律 |
| 勉强过关（刚好80分） | 险过剃头、及格万岁、压线大师 |
| 道歉次数 >= 7 | 认怂专家、滑跪大师、卑微本微 |
| 提出弥补方案 >= 5 | 画饼大师、补偿型人格、承诺大师 |
| 找借口 >= 5 | 甩锅侠、借口专家、嘴硬本硬 |
| 使用回退功能 | 反复横跳、后悔大师、选择困难症 |
| 全部选项都选过 | 究极探索者、选项收集癖、全试一遍选手 |
| 切换声音3次+ | 声优控、百变怪、变声器玩家 |

**隐藏成就**：
- 通关后随机触发
- 名字也不告诉用户，解锁时随机出现
- 候选：天选之人、欧皇认证、玄学大师

### 2.13 复盘系统

**触发时机**：游戏结束后

**复盘逻辑**：
- 将完整对话历史（10轮）发给LLM
- LLM自己分析用户的选择模式
- 生成"你这局犯的错"总结

**复盘语气**：损友 + 暖心结合
- 先吐槽用户的翻车操作（损友风格）
- 再给予鼓励和具体建议（暖心风格）
- 重点指出最薄弱的环节

**复盘示例**：
> 你这一局找了4次借口，我要是你对象早分手了。不过最后两轮选择很聪明！其实你哄人的潜力很大的，下次记住：真诚比套路管用～

### 2.14 分享卡片

**生成时机**：游戏结束后一键生成

**卡片内容**：
- 最高光时刻（LLM自动选出，本局最有意思的一轮对话）
- 通关评价（好感度级别 + 轮数级别）
- 游戏场景

**卡片风格**：精美模板，适合发朋友圈

## 3. 技术约束

### 3.1 技术栈
- 框架：Next.js
- 语言：TypeScript
- AI能力：扣子编程沙箱环境自带（LLM对话 + TTS语音）
- 全程中文

### 3.2 LLM调用模式
- **对话生成**：流式输出（打字机效果）
- **好感度计算**：LLM在每轮回复中明确给出数值变化，前端负责展示
- **选项生成**：每次调用传入选项历史，避免重复
- **复盘分析**：游戏结束后发送完整对话历史给LLM分析

### 3.3 数据规则
- 不做用户注册登录
- 不做数据持久化
- 刷新页面重新开始
- 回退功能仅在当次会话内有效

---

## 4. 明确不做的事

1. 不做自由文本输入（只用选择题）
2. 不做用户注册登录
3. 不做数据持久化（刷新页面重新开始）
4. 不做NSFW内容
5. 不做多人对战
6. 不预设题库（全部动态生成）
7. 不显示具体好感度数字
8. 奇葩选项禁止：外貌羞辱/分手离婚/提及对方家人

---

## 5. 成功标准

1. 选完场景后立刻开始，无加载等待
2. 每轮6个选项，2好4坏，坏选项里有让人笑出来的
3. 对方的回复有情绪层次，和前面对话连贯
4. 每轮对方的回复能播放语音
5. 好感度变化有动画反馈（方向提示 + 进度条）
6. 玩完后用户想截图发朋友圈或者分享链接让朋友也试试
7. 成就系统正常工作，触发时随机显示名字
8. 复盘内容具体指出用户这局犯的错
9. 分享卡片能一键生成，选取最高光时刻
10. 情绪头像随好感度变化自动切换

---

## 6. 文风规范

- 俏皮、搞笑、给人轻松感
- 对方生气时也要带点可爱，不要真的让人不舒服
- 损友风格吐槽 + 暖心风格鼓励结合
- 奇葩选项要离谱到好笑，但不越红线
